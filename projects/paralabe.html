Description

Paralabe is a Research & Development project which was offered to me as an internship subject by Jean-Yves Ramel, director of the Laboratory of Fundamental and Applied Computer Science of Tours (french acronym: LIFAT).

The starting point of this project was the Paradiit dataset, a set of small images of letters extracted from ancient books. The the main objective of this project was to be able to visualize this data into the new Cartolabe visualization tool by using differents techniques such as the Dimension Reduction and the Clustering.

Thanks to the collaboration of the Cartolabe project's team, and especially Jean-Daniel Fekete and Jonas Renault, this project fulfilled his objectives and contributed to the improvement of the Cartolabe tool.

Introduction

The goal of this project is to integrate and visualize the small images of letters extracted from ancient book with the Paradiit project in the Cartolabe tool.

In short terms, the Paralabe project makes the link between the Paradiit project and the Cartolabe project.

Project Paradiit Paralabe Cartolabe
Objectives Turn ancient books into accessible digital libraries by developing image-processing software which participates in a full processing chain, including layout analysis, text/graphics separation, Optical Character Recognition (OCR) and text transcription. Integrate all the small images of letters extracted from the Paradiit project into the Cartolabe project with dimension reduction and clustering technique. Web-based visualization system for large textual corpora. How do we proceed ?

As we mentionned in the introduction, the all process from the ancient books to the data visualisation is divided in three projects. We are going to focus on the Paralabe side of things because each of the two other projects have their own documentation.



To set things up a bit, at the beginning of this project we already have the Paradiit dataset and the Cartolabe tool is not available for the public yet, so we are working with as a developper directly on the Cartolabe Gitlab repo. The Paradiit dataset is composed of images of letters extracted from ancient books with a descriptive XML file associated to each file. We do have a part of this dataset on this repo for running the examples and for getting used to the different scripts of this project. This subset is available in the ./data/paralabe_patterns folder.

The first step of this project was to parse and export all the data from the dataset into a json file to have all necessaries informations about this dataset (such as pixels data, images base64 strings, target...) centralized in one data.json file. That's why the paradiit_to_json.py script exists.

Once done, we can manipulate the dataset correctly. The main part of this encoded dataset is an images x pixels array representing all the pixels of all the letter images where each images are 20 x 20 pixels, i.e 400 pixels (or 400 dimensions). To be able to visualize the dataset in a 2D-space, we need to reduce the dimension of this array from 400 to 2. Then, we can clusterize the data of this new low-dimensional array to have a better way of analysing this dataset by regrouping and making clusters of same letter images. So after setting up the data from the data.json file in a Bunch for an easier usage, the next process to prepare the dataset for the visualization in Cartolabe is divided in two parts : the dimension reduction and the clustering. We have detailed those methods underneath.

Dimensionality reduction

Dimension reduction is the process of reducing the number of random variables under consideration by obtaining a set of principal variables. This process can be done through :

  • ⛔️ feature selection by extracting a subset of relevant features (variables)
  • ✅ feature extraction by transforming the data in the high-dimensional space to a space of fewer dimensions.

In our case, we will consider the second case because as we want to keep all the data in the dataset and not just take a representative subset, this will be more relevant for our problem. The data transformation with the feature extraction can be divided here again in two dimensionality reduction techniques :

  • ⛔️ linear - as in principal component analysis (PCA)
  • ✅ nonlinear - as in Graph-based kernel PCA

The Nonlinear approach with Graph-based kernel PCA construct a low-dimensional data representation using a cost function that retains local properties of the data, and can be viewed as defining a graph-based kernel for Kernel PCA. This Nonlinear dimensionality reduction techniques includes manifold learning techniques such as Isomap, locally linear embedding (LLE), t-SNE or UMAP that we use in this project and that we will describe now.

UMAP

Uniform Manifold Approximation and Projection (UMAP) is a dimension reduction technique that can be used for visualisation, but also for general non-linear dimension reduction.

How it works ?

Keywords : dimension reduction, manifold, algebraic topology, topological data analysis, Riemannian geometry, Simplicial complex

As writen in the UMAP description, the algorithm is founded on three assumptions about the data

  • The data is uniformly distributed on a Riemannian manifold;
  • The Riemannian metric is locally constant (or can be approximated as such);
  • The manifold is locally connected.

From these assumptions it is possible to model the manifold with a fuzzy topological structure. The embedding is found by searching for a low dimensional projection of the data that has the closest possible equivalent fuzzy topological structure.

It is divided in two phases :

  1. constructing a fuzzy topological representation of the data in a high-dimensional space by computing the nearest neighboors of each point
  2. optimizing the low-dimensional representation to have as close a fuzzy topological representation as possible as measured by cross entropy.

ℹ️ If you want more informations, please check the really well-made and well-explained UMAP documentation - How UMAP Works. All their reasoning and their conception of the dimension reduction is described, complete and understandable.

In the Paralabe case

In the paralabe_clustering.py script, this dimension reduction is done by only one line of code.

clusterable_embedding = umap.UMAP(**param["UMAP_param"]).fit_transform(dataset.data)

In this simple line, we are defining on which dataset we want a dimension reduction and with wich UMAP parameters extracted from the configuration file config.yml.

Clustering

Clustering is the task of grouping a set of objects in such a way that objects in the same group (called a cluster) are more similar (in some sense) to each other than to those in other groups (clusters). This process will help us in data analysis to further improves the dimension reduction by changing and adapting hyper-parameters if necessary.

There is a lot of algorithm out there for the clustering, but in our case we will use a combination of a Density models algorithm (like DBSCAN) with a Hierarchical clustering algorithm. This algorithm is called HDBSCAN.

HDBSCAN

HDBSCAN - Hierarchical Density-Based Spatial Clustering of Applications with Noise. Performs DBSCAN over varying epsilon values and integrates the result to find a clustering that gives the best stability over epsilon. This allows HDBSCAN to find clusters of varying densities (unlike DBSCAN), and be more robust to parameter selection.

How it works ?

Keywords : DBSCAN, hierarchical clustering, density, spanning tree

The HDBSCAN algorithm is a extension of the well-known DBSCAN algorithm that doesn't have the following step 4 : the condensed clusters tree. It is separated into 5 steps :

  1. Transform the space according to the density/sparsity - A mutual reachability distance is defined between points to spread apart points with low density.
  2. Build the minimum spanning tree of the distance weighted graph - This is the minimum spanning tree from the mutual reachability distance.
  3. Construct a cluster hierarchy of connected components - To better see the links between components.
  4. Condense the cluster hierarchy based on minimum cluster size - For an easier interpretation of the previous hierarchy.
  5. Extract the stable clusters from the condensed tree - A cluster will be one of the greatest areas of ink in the plot. If a cluster is selected, their descendants can not be selected.

ℹ️ Once again if you want more informations, please check the equaly well-made HDBSCAN documentation - How HDBSCAN Works. Compared to the previous documentation, the algorithm has the advantage here of being explained and presented as an example through a Jupyter notebook. So, if you want to test it yourself, it's entirely possible !

In the Paralabe case

In the paralabe_clustering.py script, the clustering is also sone by only one line of code.

clusters = hdbscan.HDBSCAN(**param["HDBSCAN_param"]).fit_predict(clusterable_embedding)

Here we are telling HDBSCAN to do a clustering on the low-dimensional embedding that we created before and with the corresponding parameters extracted from the configuration file config.yml.

Conclusion

Thanks to UMAP and HDBSCAN, it's possible for us to bring a new approach on the clustering of the Paradiit project's data. After understanding what's behind these two libraries, it becomes easier to find the right hyper-parameters and improves the final results of the clustering. With the right hyper-parameters, we can have a really good clustering of all the images in a short time and with a result close to a human-hand method. In conclusion, we only have to find the perfects hyper-parameters, it's that easy ! Well, more or less...
